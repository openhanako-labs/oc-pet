# 角色边界说明

> 写于 2026-07-19。源起：ophelia 在汇报中混淆了"月曦夜"和"yuexinmiao"。

## 三个名字不要混

| 名字 | 身份 | 在桌宠里的角色 |
|---|---|---|
| **月曦夜** | 用户 | 操作桌宠的人 |
| **yuexinmiao / 月薪喵** | OC | 桌宠默认 OC 角色（`config.json` 里 `character: "yuexinmiao"`） |
| **奥菲莉娅（ophelia）** | OC / 助手 | **不是**桌宠默认角色（git 历史 `3e1096d` 移除了 ophelia 的默认 trace） |

## 边界规则

后续所有 UI / 工具 / 功能开发必须遵守：

1. **主题系统、UI 组件、活动流不绑定任何 OC 角色**
   - QSS 变量是 `light` / `dark`，跟 OC 名无关
   - 不写"奥菲莉娅 / 月薪喵专属"配色之类

2. **桌宠当前 `config.character` 是什么，UI 就显示什么**
   - 不要硬编码 OC 名
   - 涉及角色专属资源（头像 / 精灵 / 音色）按 `config.character` 读

3. **git author 不预设**
   - 提交时根据实际执行者填，不要默写"ophelia"
   - 如果助手不是 ophelia，不要用 `ophelia@hanako.local` 作为 author

4. **文档 / 汇报里区分用户和 OC**
   - "用户月曦夜" ≠ "OC 月薪喵"
   - 月薪喵的偏好不等于用户的偏好（除非显式声明）
