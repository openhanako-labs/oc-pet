# 桌宠创作规范 — 嘴型与自定义动画帧

## 概述

桌宠支持**自动发现**帧目录。你只需在 `frames/` 下创建子目录并放入 PNG 序列，系统自动加载。

本文档说明嘴型帧规范和自定义动画帧的添加方式。

---

## 一、目录结构

```
~/.hanako/agents/<角色名>/pet/
├── idle.png                    # 默认姿态（可选）
├── frames/
│   ├── idle/                   # 待机动画（必须）
│   │   ├── idle_0.png
│   │   ├── idle_1.png
│   │   ├── idle_2.png
│   │   └── idle_3.png
│   ├── walk/                   # 行走动画
│   │   ├── walk_0.png
│   │   └── ...
│   ├── extra/                  # 特殊动画（打招呼、跳跃等）
│   │   ├── extra_0.png
│   │   └── ...
│   ├── speak_closed/           # 闭嘴（静音时默认）
│   │   └── speak_closed_0.png
│   ├── speak_half/             # 小幅张嘴（低音量 / 轻声）
│   │   └── speak_half_0.png
│   └── speak_open/             # 大幅张嘴（正常说话 / 高音量）
│       └── speak_open_0.png
```

**规则：**
- 目录名即动画序列名，任意英文小写 + 下划线
- 帧文件必须是 `.png` 格式
- 文件名按字母序排列即为播放顺序（建议用 `_0`, `_1`, `_2` 后缀）
- 每个目录至少 1 帧

## 二、嘴型帧规范

### 三种嘴型状态

| 序列名 | 状态 | 触发时机 | 帧数建议 |
|--------|------|---------|---------|
| `speak_closed` | 闭嘴 | 静音、播放结束、无音频 | 1~2 帧 |
| `speak_half` | 小幅张嘴 | 低音量、轻声、TTS 低情感 | 1~2 帧 |
| `speak_open` | 大幅张嘴 | 正常说话、高音量、TTS 高情感 | 1~2 帧 |

### 帧尺寸

- 必须与 `idle` 帧尺寸一致（默认 192×208px，或你的角色实际尺寸）
- 角色身体位置保持不变，只改变嘴巴形状
- 背景透明

### 设计要点

1. **以 idle 帧为基础**——复制 idle 帧，只修改嘴巴部分
2. **嘴巴变化要明显**——缩小后仍能辨认张嘴/闭嘴
3. **三种嘴型的过渡要自然**——闭嘴 → 半张 → 全张 是渐进关系
4. **不要改变身体姿态**——只有嘴部变化

### 生成提示词参考

```
基于 idle 帧生成嘴型变体：
- 复制角色 idle 姿态
- 只修改嘴巴区域
- speak_closed: 嘴巴闭合，自然表情
- speak_half: 嘴巴微张，像在轻声说话
- speak_open: 嘴巴张开，像在正常说话
- 身体、服装、背景保持不变
- 透明背景
```

## 三、自定义动画帧

你可以创建任意名称的动画序列，系统自动发现。

### 已保留的序列名

| 序列名 | 用途 | 说明 |
|--------|------|------|
| `idle` | 待机 | 必须存在，作为默认序列 |
| `walk` | 行走 | 可选 |
| `extra` | 特殊动作 | 可选，用于打招呼/跳跃等 |
| `speak_closed` | 闭嘴 | 可选，口型系统用 |
| `speak_half` | 半张嘴 | 可选，口型系统用 |
| `speak_open` | 全张嘴 | 可选，口型系统用 |

### 自定义序列

你可以创建任意序列，例如：

```
frames/
├── sleep/          # 睡觉
├── eat/            # 吃东西
├── read/           # 看书
├── wave/           # 挥手
└── surprise/       # 惊讶
```

在代码中通过 `play_anim("sleep")` 切换到该序列。

### 帧数与帧率

| 动画类型 | 建议帧数 | 建议帧率 |
|---------|---------|---------|
| 待机（idle） | 4~6 帧 | 3 fps |
| 行走（walk） | 4~8 帧 | 4~6 fps |
| 嘴型（speak_*） | 1~2 帧 | - |
| 特殊动作 | 4~8 帧 | 4~6 fps |
| 长动画（sleep 等） | 6~12 帧 | 2~3 fps |

## 四、口型系统工作原理

```text
TTS 播放开始
  → on_start 回调触发
  → 桌宠切换到 speak_open 或 speak_half（根据音量/情感）

TTS 播放结束
  → on_end 回调触发
  → 桌宠切换回 idle

音乐播放（未来）
  → 音频事件触发
  → 同样的嘴型切换逻辑
```

### 触发条件

| 条件 | 嘴型 |
|------|------|
| TTS 开始播放 + 情感为 happy/angry | `speak_open` |
| TTS 开始播放 + 情感为 neutral/thinking | `speak_half` |
| TTS 开始播放 + 情感为 sad | `speak_half` |
| TTS 播放结束 | `speak_closed` → 回到 `idle` |
| 无 speak_* 帧目录 | 不做嘴型切换，保持当前动画 |

## 五、打包清单

```
characters/<name>/
├── pet.json
├── frames/
│   ├── idle/          ← 必须
│   ├── walk/          ← 可选
│   ├── extra/         ← 可选
│   ├── speak_closed/  ← 可选（口型用）
│   ├── speak_half/    ← 可选（口型用）
│   └── speak_open/    ← 可选（口型用）
└── preview/
    └── mouth-test.gif  ← 可选，三种嘴型的预览
```

### pet.json 中无需配置嘴型

帧目录是自动发现的。只要在 `frames/` 下创建目录并放入 PNG，系统自动加载。

---

## 六、验证

1. 启动桌宠，确认 idle 动画正常
2. 发送消息触发 TTS，观察是否切换到张嘴帧
3. TTS 结束后，确认回到 idle
4. 如果没有 speak_* 帧，确认桌宠不受影响（降级到无口型模式）
