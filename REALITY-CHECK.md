# oc-pet 功能真实性审计

> 审计日期：2026-07-08  
> 审计人：奥菲莉娅  
> 标准：不是"代码写了"，而是"用户能感知到效果"

---

## 真正在工作的功能

| 功能 | 用户能感知到什么 | 证据 |
|------|----------------|------|
| 对话 | 打字 -> 桌宠用角色口吻回复 | LLM 回复 + 气泡显示 |
| TTS 语音输出 | 桌宠用角色克隆音色说话 | CosyVoice2 + 参考音频 |
| 语音输入 | 右键"说话" -> 录音 -> 识别 -> 自动回复 | Whisper base 模型（待实测） |
| 时间感知 | LLM 回复会根据时间段变化 | bridge 注入 `[当前时间：晚上 22:30]` |
| Hanako 原生 | 桌宠用 Hanako 同一套身份/模型/API | 读 identity.md + agnes-2.0-flash |
| 桥接守护 | 后台自动收发消息 | companion_bridge.py 进程 |
| 帧精灵渲染 | 角色在桌面上动 | idle/walk/extra 动画 |
| 瞳孔跟踪 | 角色眼睛跟着鼠标转 | eye_overlay.py |
| 行为模式 | 静默/正常/活跃/黏人影响走动 | 右键菜单可切换 |
| 右键菜单 | 对话/说话/行为/缩放/角色/穿透 | 菜单项可见 |
| 鼠标穿透 | 桌宠可以"穿过去" | 右键切换 |
| 系统托盘 | 最小化到托盘 | 托盘图标 |

## 写了但基本没用的功能

| 功能 | 为什么没用 | 修复成本 |
|------|-----------|---------|
| 情绪帧区间 | 角色只有 4 张 extra 帧，映射到 [0..7] 大部分越界。情绪检测靠关键词，LLM 回复经常不触发 | 需要画更多帧 + 更好的情绪检测 |
| 情绪状态机 | 衰减逻辑写了，但结果没接到视觉表现。状态机在跑但用户看不到任何变化 | 需要接回渲染层 |
| 日程感知 | 读 `~/.hanako/.ephemeral/automation*.json`，路径基本找不到东西 | 需要确认 Hanako 自动化任务的实际存储位置 |
| 记忆压缩 | 代码完整，但要聊够 50 条才触发第一次压缩。日常使用几乎不会触发 | 降低阈值或等自然触发 |
| Avatar 抽象 | 接口定义了，SpriteRenderer 提取了，但 pet.py 还有很多地方直接访问 char_label/eye_overlay，没走接口 | 需要彻底重构 pet.py |
| Proactive 主动对话 | 代码完整，但要闲置 5 分钟 + 前台窗口匹配才触发。测试时不会触发 | 需要真实场景验证 |
| 前台窗口检测 | 在跑，但检测结果只重置 idle 计时器，没有联动对话或行为 | 需要接 proactive_scheduler |
| 动作联动 | 8 种预设动作，但需要 LLM 回复里命中关键词才高亮。实际很少触发 | 需要 LLM 配合或改为主动展示 |

## 核心链路图（实际在跑的）

```
用户
  │
  ├─ 打字 ──────────────> outbox.json
  │                         │
  ├─ 右键"说话" ──> 录音 ──> Whisper 识别 ──> outbox.json
  │                                              │
  │                                    companion_bridge.py
  │                                    ├─ 读 outbox
  │                                    ├─ 调 LLM (agnes-2.0-flash)
  │                                    ├─ 读 Hanako identity.md/ishiki.md
  │                                    ├─ 注入时间感知上下文
  │                                    ├─ CosyVoice 合成语音
  │                                    └─ 写 response.json (文本+音频)
  │                                              │
  │                                    pet (hanako_monitor)
  │                                    ├─ 读 response.json
  │                                    ├─ 显示气泡
  │                                    └─ 播放音频
  │
  └─ 桌宠窗口
      ├─ 帧精灵动画
      ├─ 瞳孔跟踪
      ├─ 行为物理（走动/弹跳）
      └─ 状态指示器
```

## 没在跑的链路

```
情绪帧区间 ❌  素材不够 + 检测不触发
情绪状态机 ❌  结果没接视觉
日程感知   ❌  读不到数据
记忆压缩   ❌  50 条阈值未达
Proactive  ❌  未在真实场景触发
Avatar 抽象 ⚠️  半成品
动作联动   ❌  关键词不命中
```

---

> 结论：核心对话+语音链路通了，其余功能大部分是"代码写了但没跑起来"。
> 下一步应该选择：补素材让现有功能生效，还是砍掉半成品减少维护负担。
